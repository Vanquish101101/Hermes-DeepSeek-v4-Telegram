# Инструкция к оркестрации 1 (Foresight)

> Подход «Foresight»: **лёгкая оркестрация** — Hermes решает задачи сам, опираясь на
> правильный выбор MCP-инструментов (поиск/исследование/скрап/память), без тяжёлого
> департамента субагентов. Альтернатива — «Инструкция к оркестрации 2 (Мой вариант)».

Версия: **1 (Foresight)**. Дата старта: 21.06.2026.
Связано: `SOUL.md` Hermes (правила), `Departments/Инструкция к оркестрации 2 (Мой вариант).md` (альтернативный алгоритм, на паузе).

---

## Идея подхода
Вместо департамента из 31 субагента — Hermes как один сильный исполнитель, который для каждой
подзадачи берёт оптимальный инструмент по типу и стоимости (приоритет MCP). Дёшево, быстро,
прозрачно. Разведка и факты — через Perplexity (ответ из живого веба с источниками).

---

## Проделанные этапы

### Этап 0 — Подключение и подтверждение Perplexity MCP (21.06.2026)
Подтверждено на всех уровнях:
- **Конфиг:** `config.yaml` → `perplexity` = `npx -y @perplexity-ai/mcp-server` (v0.9.0), `enabled: true`.
- **Hermes видит MCP:** `hermes mcp list` → `perplexity … ✓ enabled`.
- **Ключ (эмпирически):** прямой запрос к `api.perplexity.ai/chat/completions` → **HTTP 200**, ответ + живые цитаты (`last_updated` 21.06.2026), стоимость ~$0.005/запрос. Ключ `pplx-R1jb…` валиден.

### Этап 1 (пункт «а») — Роутинг инструментов поиска/исследования в SOUL.md
Добавлено **Правило 5** в `C:\Users\Unknown\AppData\Local\hermes\SOUL.md` («Роутинг инструментов поиска и исследования»):

| Задача | Инструмент |
|--------|-----------|
| Свежий факт / ресёрч + нужны источники-пруфы | **Perplexity** `sonar` / `sonar-pro` |
| Глубокое исследование, развёрнутый отчёт | **Perplexity** `sonar-deep-research` |
| Анализ / рассуждение поверх найденного | **Perplexity** `sonar-reasoning-pro` |
| Прочитать конкретный известный URL | Firecrawl / WebFetch |
| Массовый структурированный скрап | Apify |
| Документация библиотек / фреймворков | Context7 |
| Рассуждение над данным контекстом, большой контекст | Gemini / OpenRouter |
| Пульс соцсетей / X | x_search (grok) |

**Ниша Perplexity:** «вопрос → синтезированный ответ из живого веба **с источниками**». Модели по стоимости: `sonar` → `sonar-pro` → `sonar-reasoning-pro` → `sonar-deep-research` (брать минимально достаточную; для `sonar` `max_tokens` ≥ 16).

### Этап 2 (пункт «б») — Perplexity как основной инструмент OSINT/разведки
В шаге разведки (когда он применяется) основной инструмент — Perplexity MCP (`sonar`/`sonar-pro` для фактов с источниками, `sonar-deep-research` для отчёта по нише/конкурентам); дополняют Firecrawl (конкретные URL), Apify (скрап), x_search (соцсети). Использовать как вспомогательную информацию и образцы — **НЕ копирование**.
