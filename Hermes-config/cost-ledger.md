## Заметки
- Запись №1 — реальный тестовый запрос при подтверждении подключения Perplexity (вернул `cost.total_cost = 0.00501`). Первый ошибочный запрос (HTTP 400, `max_tokens`) денег не стоил — не учитываем.
- Записи №3-4 — модель `sonar-pro` (не `sonar`): запрос требовал глубины/цитат по 8 разным доменам, по правилу §9 «качество важнее экономии» при широком исследовательском запросе.
- Записи №5-9 — Firecrawl тарифицируется **кредитами из фикс-пула** (не $ за запрос), поэтому ведём отдельный счётчик, не смешивая с долларовой Σ. Запись №7 — наглядное сравнение: deprecated `/v1/extract` стоил **22 кредита**, а рекомендованный `v2/scrape json` (запись №8) — тот же результат за **5 кредитов**. Урок зафиксирован в SOUL.md §5/§9.
- Записи №10-11 — проверка OpenRouter (ключ счёт $10, остаток после теста $9.999816). Заодно вскрылся и исправлен баг: модели в SOUL.md §4 были записаны с дефисом (`claude-sonnet-4-6`) вместо точки (`claude-sonnet-4.6`) — реальный каталог `GET /api/v1/models` подтвердил правильный формат.
- Записи №12-15 — проверка `minimax/minimax-m3` (через тот же OpenRouter-ключ, без новых секретов). Реальной задачей (загадка + код) подтверждено качество. Найдена особенность: провайдер `AtlasCloud` (один из backend-ов OpenRouter для этой модели) продублировал ответ целиком в записи №14; 3 повторных запроса (записи №15) через `Together`/`Minimax` — чисто. Решение задокументировано в SOUL.md §4.5, не активировано (глобальная настройка `provider_routing`).
- Для авторитетного учёта LLM-токенов (DeepSeek/OpenRouter) использовать **Helicone** (дашборд расходов).
