# Perplexity MCP — настройка и роутинг (добавлено 2026-06-21, версия 1.2.0)

Perplexity подключён к Hermes как **MCP-сервер** для поиска/исследования с живыми источниками.
Настроен штатно, ключ был в конфиге уже при изучении проекта — подтверждён эмпирически (не на вид).

## Где настроено (файлы в `AppData\Local\hermes\`, в репо не коммитятся — содержат ключи)
1. **`config.yaml` → `mcp_servers.perplexity`**:
   ```yaml
   perplexity:
     command: npx
     args: ["-y", "..."]
     enabled: true
     env:
       PERPLEXITY_API_KEY: pplx-…   # ключ скрыт; реальный — в config.yaml
   ```

## Модели (тиры) и роутинг
Полная таблица — `SOUL.md` §5 («Роутинг инструментов поиска и исследования»). Кратко:

| Модель | Когда брать |
|---|---|
| `sonar` | простой быстрый вопрос, короткий факт |
| `sonar-pro` | нужна глубина/цитаты по нескольким темам сразу (шире контекст поиска) |
| `sonar-reasoning-pro` | сложный вопрос, требующий рассуждения над найденным |
| `sonar-deep-research` | многошаговый ресёрч-отчёт (самый дорогой тир) |

Выбор тира — по правилу §9 (`SOUL.md`): минимально достаточный уровень, но если простой тир не даёт нужной глубины — не экономить (качество важнее).

## Проверка (эмпирически)
- Первый тест (`sonar`, «capital of France», `max_tokens: 16`) → **HTTP 200**, корректный ответ, `usage.cost.total_cost` вернулась в самом ответе API.
- `hermes mcp list` → Perplexity показан как `enabled`.
- Реальный рабочий запуск: исследование «топ AI-моделей по 8 категориям» (`sonar-pro`, 2 запроса) — см. `cost-ledger.md` записи №3-4.

## Особенность учёта стоимости
В отличие от DeepSeek (требует Helicone-прокси, см. `Helicone-setup.md`), Perplexity **сам возвращает точную стоимость** в каждом ответе (`usage.cost.total_cost`) — её и заносим в `cost-ledger.md` без оценок (правило §8).

## Связанные правила/файлы
