# Firecrawl — парсер сайтов (настройка 2026-06-21, версия 1.3.0)

## Что это
Firecrawl — API/MCP-сервис, который превращает **любой сайт/страницу в чистые данные для AI**:
markdown без рекламы и вёрстки, карту всех URL сайта, структурированный JSON по заданной схеме,
полный обход (crawl) каталога/блога целиком. Решает задачу «дай мне реальный контент конкретного
ресурса», а не «найди что-то в интернете по теме» (это зона Perplexity, см. `Perplexity-setup.md`).

## Где настроено
`config.yaml` → `mcp_servers.firecrawl`:
```yaml
firecrawl:
  command: npx
  args: [-y, firecrawl-mcp]
  env:
    FIRECRAWL_API_KEY: fc-…   # ключ скрыт; реальный — в config.yaml
  enabled: true
```
Был подключён ранее, но без описания и без проверки реальных режимов — этот пробел закрывает версия 1.3.0.

## Режимы (проверены эмпирически 21.06.2026, реальными запросами к api.firecrawl.dev)

| Режим | Что делает | Стоимость | Когда брать |
|---|---|---|---|
| `scrape` | 1 страница → чистый markdown | 1 кредит | нужен текст конкретной страницы |
| `map` | сайт → список всех его URL | 0 кредитов (бесплатно) | сначала понять структуру сайта |
| `search` | веб-поиск без синтеза (просто ссылки+сниппеты) | ≈1.3 кр./результат | быстрый список релевантных страниц |
| `v2/scrape` + `formats:[{type:"json", schema/prompt}]` | страница → структурированные данные по схеме | 5 кредитов | вытащить цены/фичи/контакты в JSON |
| `crawl` | обход всего сайта/раздела (множество страниц) | по странице (как scrape × N) | весь блог/каталог целиком |
| ⚠️ `/v1/extract` | то же, что v2/scrape json, но **deprecated** | 22 кредита | НЕ использовать — дороже и устарел |

Тариф — **фикс-пул 1000 кредитов/billing-период** (не $ за запрос, в отличие от Perplexity). API не отдаёт
$-цену за кредит — точные деньги смотреть на firecrawl.dev/billing. Учёт по кредитам — `cost-ledger.md`.

## Конкретные примеры использования в нашей сборке

**1. Разведка конкурента (пайплайн Foresight, этап OSINT):**
```
map(competitor.com, limit=50)                         → список страниц сайта
  → выбрать релевантные (pricing, features, blog)
  → v2/scrape(json) по каждой со схемой {price, features[], plan_name}
  → готовая сравнительная таблица для market-competitor-analyst
```
Дешевле и точнее, чем просить LLM «вспомнить» сайт конкурента по памяти — данные настоящие, с датой.

**2. Чтение конкретного источника, который дал Perplexity:**
Perplexity вернул ссылку-цитату на статью → нужен полный текст, а не сниппет → `scrape(url)` → markdown
в контекст. Связка «Perplexity находит → Firecrawl читает целиком» дешевле, чем `sonar-deep-research`.

**3. Мониторинг изменений на странице (цена/статус конкурента):**
Периодический `scrape` той же страницы (раз в N дней через Keep-Alive-подобный cron) → сравнить markdown
с прошлым снимком → если изменилось — уведомить в Telegram. Не требует ручного открытия сайта.

**4. Быстрый список материалов по теме без дорогого синтеза:**
`search("тема", limit=5)` — когда нужны просто релевантные ссылки (а не развёрнутый ответ с цитатами,
как у Perplexity) — дешевле, чем `sonar`, если синтез не нужен, только список источников.

## Проверка (эмпирически, 21.06.2026)
- `scrape` на example.com → HTTP 200, `success: true`, 1 кредит.
- `map` на firecrawl.dev → 15 URL за один вызов, 0 кредитов.
- `search` на тестовый запрос → 3 результата (ссылка+título+описание), 4 кредита.
- `/v1/extract` → сработал, но вернул deprecation-warning + `replacement: /v2/scrape` → перепроверено
  на `v2/scrape` с `json`-форматом — тот же тип результата за 5 кредитов вместо 22.
- Баланс счёта проверен через `v1/team/credit-usage`: план 1000 кр./период (2026-06-09 → 2026-07-09).
