# Replicate MCP — настройка и архитектура генерации медиа (добавлено 2026-06-22, версия 1.11.0)

Replicate был подключён к Hermes (`config.yaml`, `enabled: true`) ещё в прошлой версии, но до этого
этапа не имел роли в оркестрации.

## Где настроено (файл в `AppData\Local\hermes\`, в репо не коммитится)
**`config.yaml` → `mcp_servers.replicate`**:
```yaml
replicate:
  command: npx
  args: ["-y", "replicate-mcp"]
  env:
    REPLICATE_API_TOKEN: r8_***************************Se
  enabled: true
```
⚠️ Особенность пакета (как у Vercel с `API_KEY`) — переменная называется именно `REPLICATE_API_TOKEN`,
не `REPLICATE_API_KEY` — задокументировано в `MCP key connection.txt`.

## Архитектура
Полное описание — `SOUL.md` §15. Кратко: Replicate — облачный слой инференса генеративных моделей
(изображения/видео/аудио/3D-текстуры), платный за вызов (в отличие от большинства уже подключённых
сервисов с фиксированным планом). Это **формализация уже распределённой практики**, а не новая
возможность: департамент CORE (`.claude/agents/`) уже называет Replicate MCP в стеке трёх специалистов:
- `54-visual-image-artist.md` — генерация изображений/иллюстраций/OG-картинок.
- `52-audio-producer.md` — музыка/звук (вместе с ElevenLabs MCP — голос).
- `24-threed-artist.md` — текстуры для 3D/WebGL-сцен.

Hermes сам (вне департамента) подключает Replicate напрямую, когда пользователь в Telegram прямо просит
сгенерировать изображение/аудио/видео и встроенная генерация Gemini не подходит.

## Проверка (эмпирически, 22.06.2026)
- `GET /v1/account` → токен валиден, аккаунт `vanquish` (`type: organization`).
- ⚠️ **Баг и фикс (важно для будущих прямых REST-вызовов к Replicate, не относится к самому MCP-пакету):**
  первый запрос с одним только `Authorization: Bearer <token>` вернул **HTTP 403, `error code: 1010`**
  (блокировка Cloudflare по сигнатуре бота — триггерится дефолтным User-Agent `urllib`). Токен был валиден
  всё это время. Фикс — добавить обычный браузерный заголовок:
  ```python
  headers = {
      "Authorization": f"Bearer {TOKEN}",
      "User-Agent": "Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36",
      "Accept": "application/json",
  }
  ```
  После этого все вызовы — HTTP 200.
- `GET /v1/predictions` → одна предсуществующая запись: `black-forest-labs/flux-1.1-pro`,
  `created_at: 2026-06-15T20:16:45Z`, `status: succeeded`, `predict_time: ≈2.76s`, `data_removed: true`
  (результат уже не скачать) — реальное использование до этого подключения, **не созданное в рамках
  этого этапа**, не трогали/не приписывали себе.
- **Реальный тест полного цикла (создан в рамках этого подключения):**
  `POST /v1/models/black-forest-labs/flux-schnell/predictions` с заголовком `Prefer: wait=30` (синхронное
  ожидание результата) → `status: succeeded` за `predict_time: 0.54s`, `total_time: 1.09s`, результат —
  готовая ссылка на изображение (`https://replicate.delivery/.../out-0.webp`). Подтверждён не только
  валидный токен, но и весь цикл запрос → генерация → результат.
- `GET /v1/account/billing` → 404 — стоимость через API недоступна, ориентироваться на `predict_time` +
  публичную таблицу цен модели (у `flux-schnell` — фиксированная ставка за изображение, одна из самых
  дёшевых моделей на платформе).
