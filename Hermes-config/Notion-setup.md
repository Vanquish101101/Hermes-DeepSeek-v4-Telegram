# Notion MCP — настройка и архитектура базы знаний (добавлено 2026-06-22, версия 1.6.0)

Notion подключён к Hermes как **MCP-сервер** ещё в версии 0.0.5 (15.06.2026), но до этой версии не имел
роли в оркестрации — токен был рабочим, но ни одного правила «когда использовать» под него не было.

## Где настроено (файл в `AppData\Local\hermes\`, в репо не коммитится — содержит ключ)
**`config.yaml` → `mcp_servers.notion`**:
```yaml
notion:
  command: npx
  args: ["-y", "@notionhq/notion-mcp-server"]
  enabled: true
  env:
    NOTION_TOKEN: ntn_…   # ключ скрыт; реальный — в config.yaml
```
Прямое подключение от Notion (Integration token), без посредников типа MCP Market.

## Архитектура базы знаний
Полное описание правил — `SOUL.md` §10. Кратко:
- Корневая страница `Foresight — База знаний (Hermes)` (workspace-level).
- База данных `Исследования и находки` — свойства `Name` / `Дата` / `Инструмент` / `Категория` / `Стоимость ($)` / `Версия проекта`.
- Роль: контентный архив **значимых** находок (не каждый мелкий запрос) — отдельно от `cost-ledger.md` (финансы, §8) и `SOUL.md` (правила).
- Перед новым платным исследованием — сначала `search` в этой базе (экономия, §9, §5).

## Проверка (эмпирически, 22.06.2026)
- `POST /v1/search` → HTTP 200, токен валиден; видно 20 объектов workspace (включая чужую неотносящуюся
  структуру «🤖 ИИ Департамент — Регламенты и Стандарты работы» — **не трогаем**, не наш контент).
- `POST /v1/pages` с `parent: {"type": "workspace", "workspace": true}` → создана корневая страница
  `Foresight — База знаний (Hermes)`.
- `POST /v1/databases` (parent = корневая страница) → создана база «Исследования и находки» с 6 свойствами.
- `POST /v1/pages` (parent = база) → создана первая запись — находка AtlasCloud/MiniMax M3 (см. `SOUL.md`
  §4.5), с телом-описанием в 2 параграфа.
- Заголовок проверен побайтово: `title.encode('utf-8').hex()` совпал с ожидаемым значением. Первая попытка
  создать страницу через `curl` с JSON в heredoc дала битую кодировку кириллицы (mojibake) — печать в
  терминале и реальные данные совпадали лишь после перехода на Python `urllib.request` с явным
  `json.dumps(..., ensure_ascii=False).encode("utf-8")`. Вывод: для записи кириллицы в Notion API через
  этот Bash-инструмент использовать Python, не `curl` напрямую.

## Особенность: создание top-level страниц
Notion API обычно не позволяет внутренним интеграциям создавать страницы на уровне workspace — но для
этого конкретного integration token запрос с `parent.workspace: true` прошёл (HTTP 200, не 403/404).
Если понадобится ещё один корневой раздел — пробовать тот же способ; если откажет — создавать как child
существующей страницы `Foresight — База знаний`.
