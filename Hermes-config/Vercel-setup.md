# Vercel MCP — настройка и архитектура статуса деплоя (добавлено 2026-06-22, версия 1.9.0)

Vercel подключён к Hermes как community MCP (пакет `@open-mcp/vercel`) ранее, но до этой версии
не имел роли в оркестрации.

## Где настроено (файл в `AppData\Local\hermes\`, в репо не коммитится — содержит ключ)
**`config.yaml` → `mcp_servers.vercel`**:
```yaml
vercel:
  command: npx
  args: ["-y", "@open-mcp/vercel"]
  enabled: true
  env:
    API_KEY: vcp_…   # ключ скрыт; реальный — в config.yaml
```
⚠️ Особенность этого community-пакета: переменная окружения называется просто `API_KEY`,
не `VERCEL_API_KEY` (как можно было бы ожидать по аналогии с другими сервисами) — задокументировано
в `MCP key connection.txt`.

**Fallback**, если community-пакет однажды откажет (community-пакеты менее стабильны, чем official):
официальный OAuth-вариант не требует нового ключа — другой механизм авторизации:
```
npx -y mcp-remote https://mcp.vercel.com
```

## Архитектура
Полное описание — `SOUL.md` §13. Кратко: Vercel — слой **статуса деплоя и наблюдаемости**, не слой
действия. В отличие от PostMyPost (§11, публикация с подтверждением пользователя), Vercel в первую
очередь читает состояние (проекты/деплои/домены/логи) и докладывает пользователю в Telegram —
запись (новый деплой, смена alias главного домена) делается отдельно, осторожно и только по прямой
команде.

### ⚠️ Жёсткое ограничение
Никаких изменений прод-окружения без явной команды пользователя — push в GitHub не деплоит
автоматически, новый деплой по умолчанию preview (без `--prod`), несколько версий могут жить
одновременно. Правило перенесено из отдельного проекта лендинга CORE (`feedback_vercel_dont_overwrite_prod`
в памяти) и закреплено как общее правило Hermes для любого проекта на Vercel.

## Связанные правила/файлы
- `SOUL.md` §13 (архитектура), §10 (Notion — куда заносить значимые находки из логов деплоя),
  §12 (YouGile — куда заносить задачу, если деплой упал и требует ручного вмешательства).
- API-документация: https://vercel.com/docs/rest-api (REST, тот же, что использует MCP-пакет).
- Отдельная память о правиле «не перезаписывать прод без команды» — из проекта лендинга CORE,
  применяется здесь как общее правило, не специфичное для одного проекта.
