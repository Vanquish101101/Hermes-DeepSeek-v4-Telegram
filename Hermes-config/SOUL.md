## 1. Трассировка и диагностика LLM-вызовов (LangSmith)

LangSmith подключён в `config.yaml` ранее (`langsmith-mcp-server` через `uvx`, ключ `LANGSMITH_API_KEY`),
но не имел роли в оркестрации. На этом этапе подключение проверено эмпирически и сервис получил роль:
**слой трассировки и диагностики содержимого LLM-вызовов** — пошаговый разбор «промпт → вызовы
инструментов → ответ модели» для конкретного запроса. Ниша отличается от двух похожих сервисов из
удалённой оркестрации — Helicone (источник истины по $ и токенам) и Sentry (мониторинг ошибок уровня
кода, что упало в приложении) — оба описаны только в архиве, не здесь. LangSmith отвечает на третий
вопрос — «почему именно ТАК рассуждала/выбирала инструменты модель в конкретном вызове».

### 1.1 Что подключено (проверено эмпирически 23.06.2026)
- REST API `https://api.smith.langchain.com`, авторизация — заголовок `x-api-key`. Cloudflare-бага
  (403 без браузерного User-Agent, известная по Replicate/Sentry — см. архив) не было — проверяли
  защитно с первого запроса.
- `/info` → версия `0.16.6`, `instance_flags.trace_tier_duration_days: {longlived: 400, shortlived: 14}`
  (срок хранения трейсов зависит от тарифа проекта).
- `/sessions` (так LangSmith называет проекты трассировки) → один предсуществующий проект
  **«foresight»** (id `42ed25a9-a375-4953-ae26-48dc3fbb521f`, создан **08.06.2026** — раньше начала
  этой оркестрации, не результат текущей сессии), `trace_tier: "shortlived"` (14 дней хранения).
- `/runs/query` по этому проекту → **0 трассированных вызовов** — проект создан, но ни один реальный
  LLM-вызов туда никогда не логировался.
- `/datasets` → **0** — функциональность офлайн-оценки (eval) не использовалась.
- `/orgs/current` → организация **«Personal»**, **бесплатный тариф** (`plan_tier: "free"`,
  `engine_lcu_spend_limit_monthly: 0.0`, 1 identity, 1 workspace).

### 1.2 Архитектура — когда использовать
1. **Автоматической трассировки сейчас нет** — вызовы LLM не льются в LangSmith сами по себе: для
   этого нужна отдельная интеграция в код Hermes (например `LANGCHAIN_TRACING_V2`/явный SDK-вызов),
   которая не входит в это подключение.
2. Использовать вручную при расследовании конкретного подозрительного ответа/выбора инструмента модели.
   LangSmith — место, куда можно было бы затащить трейс конкретного вызова для разбора шаг-за-шагом.
3. Datasets/eval (сейчас 0) — задел на будущее для систематического тестирования качества ответов.
4. Только чтение по умолчанию — не создавать/не удалять проекты или датасеты без явной причины.

### 1.3 Связь с другими сервисами
- **Helicone / Sentry** — комплементарные роли существовали в удалённой оркестрации (см. архив
  `Инструкция к оркестрации 3`) — Helicone считал $/токены, Sentry мониторил код-исключения; здесь они
  больше не описаны как живые §, только как историческая ссылка.

> Подтверждено эмпирически (23.06.2026): REST API отвечает (версия `0.16.6`), ключ валиден. Найден один
> предсуществующий проект «foresight» (создан 08.06.2026), 0 трассированных вызовов, 0 датасетов.
> Аккаунт на бесплатном тарифе (org «Personal», лимит расходов на движок $0). Автоматической

## 2. Графовые агентные workflow и деплой (LangGraph)

LangGraph — продукт того же разработчика, что LangSmith (LangChain Inc.), но принципиально другого
типа: не SaaS-API на каждый вызов, а **(1)** open-source библиотека для построения графовых агентных
workflow с состоянием/циклами/условными переходами, исполняемая прямо в коде, и **(2)** платный
хостинг для деплоя таких графов («LangGraph Platform», в API называется «LangSmith Deployment»),
управляемый через отдельный control-plane API. Ключи `LANGGRAPH_API_KEY`/`LANGGRAPH_SERVICE_KEY` были
добавлены в `.env` Hermes ещё 15.06.2026 (версия 0.0.2, CHANGELOG.md), но без роли в оркестрации. Этот
этап даёт LangGraph роль и честно фиксирует находку: hosted-слой заблокирован тарифом.

### 2.1 Что проверено эмпирически (23.06.2026)
- Через документацию LangChain найден точный control-plane API: база `https://api.host.langchain.com`
  (регион GCP US по умолчанию), авторизация — заголовок `X-Api-Key` (валидный LangSmith-ключ) +
  `X-Tenant-Id` (id рабочего пространства).
- `GET /v2/deployments` с обоими ключами (`LANGGRAPH_API_KEY` — personal-токен `lsv2_pt_`,
  `LANGGRAPH_SERVICE_KEY` — service-токен `lsv2_sk_`), с `X-Tenant-Id` (тот же tenant, что у LangSmith,
  §1.1) и без него → во всех случаях **HTTP 403**: `{"detail":"LangSmith Deployment is not enabled
  for this organization"}`.
- Это **не проблема ключа** (оба валидного формата, сервер принял заголовки и содержательно ответил,
  не 401 «invalid token») — это тарифный гейт: фича «LangSmith Deployment» (= LangGraph Platform)
  недоступна на бесплатном тарифе org «Personal» (подтверждено в §1.1: `plan_tier: "free"`). 🟡-блокер
  в реестре багов — нужен платный план LangSmith (Plus и выше), не просто настройка.

### 2.2 Архитектура — два независимых слоя
1. **LangGraph (open-source библиотека, бесплатно, без API-вызовов)** — фреймворк для построения
   собственных агентных графов с состоянием прямо в коде. Сейчас **не используется** в коде Hermes
   (нет признаков в зависимостях/конфиге) — фиксируем как доступную архитектурную опцию на будущее, а
   не активный слой. Потенциальная польза: если когда-нибудь понадобится формализовать многошаговую
   логику самого Hermes (а не просто подключить внешний инструмент) — LangGraph даёт код-уровневую
   альтернативу департаменту субагентов («Инструкция к оркестрации 2», которая на паузе). Это два
   разных механизма (графовый код в самом Hermes vs делегирование Claude Code субагентам в этом
   репозитории) — не путать и не сливать.
2. **LangGraph Platform / «LangSmith Deployment» (платный хостинг деплоя графов)** — control-plane
   API технически подтверждён (правильный хост/заголовки/формат ключей), но **полностью заблокирован
   тарифом** — пока план не оплачен/не включён, слой недоступен независимо от того, написан ли граф.

### 2.3 Связь с другими сервисами
- **LangSmith (§1)** — общая инфраструктура: тот же tenant/org, тот же тип ключа (`lsv2_*`). Если
  когда-нибудь Platform-слой включат и логику Hermes реально перепишут графом — трассировка
  автоматически потечёт в уже существующий пустой проект «foresight» (§1) — естественно закрывает
  разрыв «MCP подключён, но трейсов нет», отмеченный в прошлой версии.
- **Инструкция к оркестрации 2 (департамент)** — концептуально похожая идея (многошаговая декомпозиция
  задачи), но другой механизм (графовый код в самом Hermes vs делегирование Claude Code субагентам) —
  не заменяет и не активирует друг друга, статусы независимы.
- **cost-ledger.md / реестр нужной оплаты** — фича заблокирована тарифом, не ценой за вызов — занесено
  в реестр нужной оплаты, а не в баги как технический дефект.

> Подтверждено эмпирически (23.06.2026): control-plane API LangGraph Platform (`api.host.langchain.com`)
> нашёлся и принял оба ключа технически правильно, но вернул `403 "LangSmith Deployment is not enabled
> for this organization"` — бесплатный тариф org «Personal» не включает деплой графов. Библиотечный
> (открытый) слой LangGraph в коде Hermes не используется — роль зафиксирована как архитектурная опция
> на будущее. Детали — `LangGraph-setup.md`.
