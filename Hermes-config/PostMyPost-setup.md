# PostMyPost MCP — настройка и архитектура публикации (добавлено 2026-06-22, версия 1.7.0)

PostMyPost подключён к Hermes как официальный MCP ещё в версии 0.3.x, но до этой версии не имел роли
в оркестрации — ключ был рабочим, но ни одного правила «когда использовать» под него не было.

## Где настроено (файл в `AppData\Local\hermes\`, в репо не коммитится — содержит ключ)
**`config.yaml` → `mcp_servers.postmypost`** — Remote HTTP:
```yaml
postmypost:
  url: https://mcp.postmypost.io/mcp
  enabled: true
  headers:
    Authorization: Bearer alZ_…   # ключ скрыт; реальный — в config.yaml
```
Транспорт — MCP Streamable HTTP (JSON-RPC 2.0 поверх обычного POST), **без сессий** — `initialize` не
возвращает `Mcp-Session-Id`, каждый вызов `tools/call` независим и не требует повторного `initialize`.

## Архитектура
Полное описание — `SOUL.md` §11. Кратко: PostMyPost — слой **публикации/дистрибуции** (готовый контент →
конкретный канал соцсети → аналитика), в паре с Notion (лог опубликованного) и противоположность
Perplexity/Firecrawl (те добывают информацию, PostMyPost — раздаёт готовое).

## Проверка (эмпирически, 22.06.2026)
- `initialize` → HTTP 200, `serverInfo.name = "postmypost-mcp-server"`, протокол `2025-11-25`.
- `tools/list` → подтверждены все 15 инструментов из документации подключения.
- `list_projects` → один проект **«Личный проект»** (`id 347765`, `timezone_id 447`).
- `list_channels` → каталог 23 платформ (Telegram, Instagram, Twitter/X, Facebook, LinkedIn, TikTok,
  Discord, Reddit, Bluesky, Mastodon, Threads и др.) — справочник кодов, не список подключённых аккаунтов.
- `list_timezones` → каталог часовых поясов (558 записей, постранично).

### ⚠️ Найден блокер: тариф истёк
`list_accounts(project_id=347765)` и `list_rubrics(project_id=347765)` → ошибка
`Internal error: Срок действия вашего тарифа истёк`. Реальные подключённые соцсети-аккаунты сейчас не
видны через API — значит, и `create_publication` сейчас не сработает. Подключение/токен/MCP — рабочие,
заблокирована именно платная функция самого PostMyPost (нужно продлить тариф на их стороне). Текст ошибки
проверен побайтово (UTF-8 hex), не на глаз — терминал искажает кириллицу визуально (тот же урок, что
и в `Notion-setup.md`).

## Что делать перед первой реальной публикацией
1. Проверить `list_accounts(project_id=347765)` — если ошибка про тариф ушла, значит можно продолжать.
2. Выбрать аккаунт/канал, при необходимости рубрику (`list_rubrics`).
3. Медиа (если есть) — `init_upload` → `complete_upload`/`status_upload`.
4. **Показать пользователю** текст+канал+время и получить явное подтверждение перед `create_publication`
   (публикация в реальную соцсеть видна другим и трудно обратима).
5. Записать результат в Notion (см. ниже).
