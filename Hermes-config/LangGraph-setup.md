# LangGraph — настройка и архитектура графовых агентных workflow (добавлено 2026-06-23, версия 1.15.0)

Ключи `LANGGRAPH_API_KEY`/`LANGGRAPH_SERVICE_KEY` были добавлены в `.env` Hermes ещё 15.06.2026
(версия 0.0.2, см. `CHANGELOG.md`), но без роли в оркестрации. На этом этапе подключение проверено
эмпирически и сервис получил роль — с честной фиксацией находки: hosted-слой заблокирован тарифом.

## Где настроено (файл в `AppData\Local\hermes\`, в репо не коммитится)
**`.env`**:
```
LANGGRAPH_API_KEY=lsv2_pt_***       # personal-токен
LANGGRAPH_SERVICE_KEY=lsv2_sk_***   # service-токен
```
Оба ключа — того же формата (`lsv2_*`), что ключ LangSmith (`Hermes-config/LangSmith-setup.md`) —
LangGraph Platform авторизуется через LangSmith API-ключи (общая инфраструктура аккаунта).

## Архитектура
Полное описание — `SOUL.md` §19. LangGraph — продукт принципиально другого типа, чем все остальные
сервисы оркестрации: не SaaS-API на каждый вызов, а **(1)** open-source библиотека для построения
графовых агентных workflow с состоянием/циклами/условными переходами, исполняемая прямо в коде, и
**(2)** платный хостинг для деплоя таких графов («LangGraph Platform», в API — «LangSmith Deployment»).

## Проверка (эмпирически, 23.06.2026)
- Через документацию LangChain (`docs.langchain.com/langgraph-platform/api-ref-control-plane`,
  `docs.langchain.com/langsmith/api-ref-control-plane`) найден точный control-plane API:
  - База: `https://api.host.langchain.com` (GCP US, дефолтный регион для облачного SaaS).
  - Авторизация: заголовок `X-Api-Key` (валидный LangSmith-ключ) + `X-Tenant-Id` (id рабочего
    пространства — использован тот же tenant, что у LangSmith, `1836e4a9-a47c-40fb-8f67-1968c8a81047`).
- `GET /v2/deployments` протестирован 3 раза: `LANGGRAPH_API_KEY` без `X-Tenant-Id`, тот же ключ с
  `X-Tenant-Id`, и `LANGGRAPH_SERVICE_KEY` с `X-Tenant-Id` → **во всех трёх — HTTP 403**:
  ```json
  {"detail":"LangSmith Deployment is not enabled for this organization"}
  ```
- Это **не проблема ключа** — оба валидного формата, сервер принял запрос и содержательно ответил
  (не `401 invalid token`, как было бы при сломанном ключе). Это тарифный гейт: фича «LangSmith
  Deployment» недоступна на бесплатном тарифе org «Personal» (подтверждено в `LangSmith-setup.md`:
  `plan_tier: "free"`).
- Cloudflare-бага User-Agent (как у Replicate/Sentry) не возникло — проверяли с браузерным заголовком
  с первого запроса.

## Ключевая находка: новый 🟡-блокер (тарифный, не технический)
В отличие от Sentry/LangSmith (где сервис подключён, но просто не интегрирован в живой код), здесь
сервис **физически недоступен на текущем тарифе** — не вопрос интеграции, а вопрос оплаты плана
LangSmith (Plus и выше включает LangGraph Platform deployments). Добавлено в реестр нужной оплаты
(`project_balance_funding_tracker`) и в реестр багов (`project_known_bugs_registry`, как блокер с
указанием причины — тариф, не код).

## Архитектура — два независимых слоя
1. **LangGraph (библиотека, бесплатно)** — НЕ используется в коде Hermes сейчас. Зафиксирован как
   архитектурная опция на будущее: код-уровневая альтернатива департаменту субагентов («Инструкция к
   оркестрации 2», на паузе) для формализации многошаговой логики самого Hermes. Разные механизмы —
   не путать и не сливать.
2. **LangGraph Platform / «LangSmith Deployment» (платный хостинг)** — control-plane API технически
   подтверждён (правильный хост/заголовки/формат ключей), но полностью заблокирован тарифом.

## Конкретная связь сервисов
- **LangSmith** — общая инфраструктура (тот же tenant/org, тот же тип ключа). Если Platform-слой
  когда-нибудь включат и логику Hermes перепишут графом, трассировка автоматически потечёт в уже
  существующий пустой проект «foresight» (`LangSmith-setup.md`) — закрывает разрыв «подключён, но
  трейсов нет».
- **Инструкция к оркестрации 2** — концептуально похожая идея (многошаговая декомпозиция), но другой
  механизм (графовый код в Hermes vs делегирование Claude Code субагентам) — статусы независимы.

## Связанные правила/файлы
- `SOUL.md` §19 (архитектура), §18 (LangSmith — общая инфраструктура и tenant).

- `project_balance_funding_tracker.md` / `project_known_bugs_registry.md` — новая запись (тарифный
